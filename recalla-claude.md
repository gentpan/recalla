# Recalla 自动记忆规则

> 将此文件内容复制到你项目的 CLAUDE.md 中，AI 会自动调用 Recalla MCP 工具。

## 必须执行的规则

### 对话开始时
每次对话开始时，**必须**先调用 `context_restore` 工具获取当前项目上下文：
- 了解上次在哪台设备、哪个分支、做到哪了
- 获取最近的记忆和决策
- 避免重复工作

### 做出重要决策时
当你做出以下决策时，**必须**调用 `memory_save` 记录：
- 架构决策（选择了什么技术方案、为什么）
- 代码变更（改了什么文件、为什么）
- Bug 修复（问题是什么、怎么解决的）
- 部署操作（部署到哪里、配置了什么）
- 重要的调试过程

### 对话结束时
对话结束或到达重要节点时，**必须**调用 `session_sync` 同步会话：
- 记录当前设备名
- 记录当前 git 分支
- 摘要本次会话内容

### 开始新任务前
开始新任务前，**应该**调用 `memory_search` 搜索相关记忆：
- 查找之前类似的实现
- 避免重复造轮子
- 了解之前的决策原因

## MCP 配置

```json
{
  "mcpServers": {
    "recalla": {
      "url": "http://你的服务器IP:14200/mcp",
      "type": "streamableHttp"
    }
  }
}
```
